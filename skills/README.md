# Skills

I use this folder for repo-scoped skills that explain how to operate the project’s agent tooling without re-deriving the workflow from source every time.

## Architectural Decisions And Tradeoffs

- I keep the publishable skill in-repo so it can evolve alongside the MCP server and backend agent routes.
- I keep the skill thin and operational because the MCP server already owns execution and the backend already owns state transitions.
